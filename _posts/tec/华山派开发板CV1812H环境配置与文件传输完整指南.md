## 1. 环境准备

### 硬件

- **开发板**：华山派（SOPHON CV1812H）
- **宿主机**：Windows 10/11

### 软件版本

| 软件 | 版本 |
|---|---|
| VMware Workstation | 16.x / 17.x |
| Ubuntu | 20.04 LTS（推荐） |
| SDK | cvi_mmf_sdk（CV1812H） |

> ⚠️ **重要提示**：SDK 官方推荐 Ubuntu 22.04，但 20.04 也可用。**不要使用 Ubuntu 24.04/26.04 等新版本**，会导致编译失败。

## 2. Ubuntu 虚拟机基础配置

### 2.1 VMware Tools 安装

```bash
# 更新软件源
sudo apt update

# 安装 open-vm-tools
sudo apt install open-vm-tools open-vm-tools-desktop -y

# 重启生效
sudo reboot
```

### 2.2 共享文件夹配置

**VMware 设置**：

1. 虚拟机 → 设置 → 选项 → 共享文件夹
2. 选择「总是启用」
3. 添加 Windows 上的文件夹，命名如 shared

**Ubuntu 挂载**：

```bash
# 创建挂载点
sudo mkdir -p /mnt/hgfs

# 挂载共享文件夹
sudo vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other

# 验证
ls /mnt/hgfs/
```

**开机自动挂载**（编辑 `/etc/fstab`）：

```
.host:/ /mnt/hgfs fuse.vmhgfs-fuse allow_other,uid=1000,gid=1000,umask=022 0 0
```

### 2.3 SSH 服务配置

```bash
# 安装 SSH 服务
sudo apt install openssh-server -y

# 启动并设置开机自启
sudo systemctl start ssh
sudo systemctl enable ssh

# 查看 IP 地址
hostname -I
```

**Windows 免密登录**（在 Windows PowerShell 执行）：

```powershell
# 生成密钥（如果没有）
ssh-keygen -t rsa -b 4096

# 复制公钥到 Ubuntu
type C:\Users\你的用户名\.ssh\id_rsa.pub | ssh 用户名@Ubuntu_IP "cat >> ~/.ssh/authorized_keys"
```

### 2.4 C 开发环境安装

```bash
sudo apt update
sudo apt install build-essential gdb manpages-dev -y

# 验证
gcc --version
```

### 2.5 Oh My Zsh 配置

```bash
# 安装 Zsh
sudo apt install zsh git curl -y

# 安装 Oh My Zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# 安装插件
cd ~/.oh-my-zsh/custom/plugins/
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git
git clone https://github.com/zsh-users/zsh-autosuggestions.git

# 编辑 ~/.zshrc，修改 plugins=(git zsh-autosuggestions zsh-syntax-highlighting)

# 生效
source ~/.zshrc
```

## 4. SDK 编译环境配置

### 4.1 下载并解压 SDK

```bash
# 进入工作目录
cd ~/huashan/workplace/sophpi-huashan/

# 下载 SDK（根据实际地址）
./download_host-tools.sh

# 解压
tar -xvf host-tools.tar.gz
```

### 4.2 环境变量配置

```bash
# 进入 SDK 目录
cd cvi_mmf_sdk

# 加载环境
source build/cvisetup.sh

# 设置板卡配置
export BOARD=cv1812h_wevb_0007a_emmc_huashan
export CHIP=cv1812h
export CHIP_ARCH=mars
```

### 4.3 配置板卡

由于 SDK 脚本存在 bug，需要手动修复。

**修复 `build/common_functions.sh` 中的 defconfig 函数**：

编辑 `build/common_functions.sh`，找到 `defconfig()` 函数，修改为：

```bash
function defconfig()
{
  local chip_arch
  local board

  board=$1
  chip_arch=`"${BUILD_PATH}/scripts/boards_scan.py" --get-chip-arch --board_name ${board}`

  if [ "${chip_arch}" != "" ]; then
    # 直接调用 python3 绕过 bug
    python3 "${BUILD_PATH}/scripts/defconfig.py" "${board}"
  else
    "${BUILD_PATH}/scripts/boards_scan.py" --list-boards ${board}
  fi
}
```

### 4.4 生成配置文件

```bash
# 复制板卡配置
cp build/boards/mars/cv1812h_wevb_0007a_emmc_huashan/cv1812h_wevb_0007a_emmc_huashan_defconfig .config
cp .config build/.config

# 创建 Kconfig 依赖
mkdir -p output build/output
touch output/Kconfig output/Kconfig.sensors output/Kconfig.panels
touch build/output/Kconfig build/output/Kconfig.sensors build/output/Kconfig.panels

# 生成配置
python3 build/scripts/gen_sensor_config.py
python3 build/scripts/gen_panel_config.py
python3 build/scripts/gencmakeconfig.py .config
```

### 4.5 编译（可选）

```bash
# 加载环境
source build/cvisetup.sh
export BOARD=cv1812h_wevb_0007a_emmc_huashan
export CHIP=cv1812h
export CHIP_ARCH=mars

# 开始编译（耗时约 30-60 分钟）
build_all
```

## 5. 网络配置与文件传输

### 5.1 TFTP 服务端配置（Windows）

1. **下载 Tftpd64**：从官网下载并安装
2. **配置**：
   - `Current Directory`：选择存放文件的文件夹
   - `Server interfaces`：选择电脑的 IP 地址（如 `192.168.150.3`）
3. **防火墙**：确保 Windows 防火墙放行 Tftpd64

### 5.2 开发板网络配置

```bash
# 查看 IP 地址
ifconfig

# 设置静态 IP（如需要）
ifconfig eth0 192.168.150.100 netmask 255.255.255.0 up
```

### 5.3 连通性测试

```bash
# 开发板 ping 电脑
ping 192.168.150.3

# 如果通，说明网络正常
```

### 5.4 文件传输

**从电脑下载文件到开发板**：

```bash
# 进入可写目录（/tmp 或 /mnt/data）
cd /mnt/data

# 下载文件
tftp -g -r helloworld_musllibc 192.168.150.3

# 查看文件
ls -la

# 添加执行权限并运行
chmod +x helloworld_musllibc
./helloworld_musllibc
```

**从开发板上传文件到电脑**：

```bash
tftp -p -l 开发板上的文件名 电脑IP
```

### 5.5 TFTP 常用命令参数

| 参数 | 含义 |
|---|---|
| `-g` | Get，从服务器下载文件 |
| `-p` | Put，上传文件到服务器 |
| `-r 文件名` | Remote，远程文件名 |
| `-l 文件名` | Local，本地文件名 |

## 6. 常见问题汇总

### Q1：编译时提示 Kconfig not found

**原因**：缺少 Kconfig 依赖文件

**解决**：

```bash
mkdir -p output build/output
touch output/Kconfig output/Kconfig.sensors output/Kconfig.panels
touch build/output/Kconfig build/output/Kconfig.sensors build/output/Kconfig.panels
python3 build/scripts/gencmakeconfig.py .config
```

### Q2：编译提示 no fip version

**原因**：`.config` 缺少 FIP 配置

**解决**：

```bash
echo "CONFIG_FIP_V2=y" >> .config
cp .config build/.config
```

### Q3：TFTP 传输失败 Read-only file system

**原因**：当前目录只读

**解决**：切换到 `/tmp` 或 `/mnt/data` 等可写目录

### Q4：TFTP 连接不上

**检查清单**：

1. 电脑防火墙是否放行 Tftpd64
2. Tftpd64 的 `Server interfaces` 是否选择了正确的 IP
3. 开发板和电脑是否在同一个网段
4. 能否互相 ping 通

### Q5：SSH 连接提示 REMOTE HOST IDENTIFICATION HAS CHANGED

**原因**：虚拟机重装后 SSH 指纹变化

**解决**（Windows）：

```powershell
# 删除旧指纹
ssh-keygen -R 192.168.124.128
# 或直接编辑 C:\Users\用户名\.ssh\known_hosts 删除对应行
```
